发表于 2018-08-28 | 分类于 框架

基于annotation的ssh框架整合配置

[TOC]

需要的配置文件

Strtsu2框架：src/strtus.xml
Spring框架：src/applicationContext.xml
关于日志：log4j.properties
关于数据库连接：db.properties

web.xml配置(同xml)

domain下bean的注解配置

  • @Entity：定义实体类
  • @Table：定义表
  • @Id：主键
  • @GeneratedValue：注解生成策略
  • @Column：定义列(可以省略)

整合hibernate

applicationContext.xml进行配置

  1. 需要引入db.properties文件，并且数据库相关信息都在db.properties文件中进行配置

  2. 配置连接池

  3. 开启注解扫描

  4. 配置sessionFactory

    1. 加载上述步骤配置的连接池
    2. 配置hibernateProperties
    3. 加载注解类

  5. 声明事务管理器

  6. 加载事务注解驱动

bean的注册
@Component：不确定位置
@Rpository：用于DAO层
@Service：用于service层
@Controller：用于表现层

dao注解配置

  1. @Repository(“userDao”)：注册bean
  2. 注入sessionFactory

service注解配置

  1. @Service(“userService”)：注册bean // 注意：要定义名字，否则在action注入时可能报错
  2. @Transactional：开启事务
  3. 注入dao：需要提供set方法

action注解配置

==注意==：

1.需要struts2-spring-plugin.jar
2.action类需要继承ActionSupport;
3.数据封装：实现ModelDriven<T>接口并提供实例化bean对象

  1. 类上基本配置
    @Controller
    @Scope(“prototype”)
    @Namespace(“/“)
    @ParentPackage(“struts-default”)

  2. service注入

  3. 配置action

延迟加载的问题

在web.xml中配置openSessionInViewFilter
==注意:openSessionInViewFilter一定要在Struts2 Filter前配置==

方式二：spring管理hibernate配置(建议使用)

该方式不需要hibernate.cfg.xml文件，所有关于hibernate.cfg.xml文件中的配置都在spring的配置文件中来配置
